Scripting & Triggers

Learn how to use variables and triggers to add complex logic to your gametype.

Variables

Variables are used to store and manipulate state. Each variable can be local, networked (periodically sent to all players), or networked_high (sent more frequently).

variables global
    local number global_number 0
    networked timer global_timer 10
    local team global_team defenders
    local player global_player none
    local object global_object none
end

variables player
    local number is_zombie false
end

Triggers

Triggers are the core of Megalo scripts. They execute their contained code every tick if their conditions are met. Triggers are processed from top to bottom.

  • General: Runs once per tick.
  • Player: Loops through every player per tick. The current player is referenced with current_player.
  • Object: Loops through every object per tick. The current object is referenced with current_object.
  • Initialization: Runs once at the beginning of the match.
  • Object Death: Runs once every time certain objects are destroyed. Use object_death_damage_type, object_death_dead_object, object_death_killing_object, and object_death_killing_player.

Sub-triggers (For-each Loops)

You can use the for_each action to loop through players, objects, and map objects inside any trigger.

trigger general
    temporary number player_count 0
    action for_each player
        action set player_count += 1
    end
end
