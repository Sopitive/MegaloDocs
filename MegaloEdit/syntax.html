MegaloEdit Syntax & Language

Learn the core features of the MegaloEdit language, from comments and autocomplete to string tables and includes.

General Features

Comments
Use the semicolon (;) to treat everything to the right as a comment. Note that autocomplete may still trigger even inside comments if you use a period.
Autocomplete
Press Ctrl + Space to bring up the autocomplete dropdown. While not perfect in all Halo variants (especially H4 & H2A), it is indispensable for exploring available actions and properties.
Error Box
The error box is located at the bottom of the window. If it doesn't explicitly start with "WARNING", assume the error will prevent building your gametype.

String Tables

Strings are lines of text that can be referenced elsewhere. It is best practice to define them in a dedicated table, often in a separate file.

string_table english 
    name_of_string "content of string"
    engine_name "Example Gametype"
    engine_description "This is an example description."
end

Constants

Constants allow you to assign a name to a number for easy reference. By convention, constants start with a "k", but this is optional.

constants
    number k_enabled 1
    number k_disabled 0
    number k_score_limit 777
end

Includes & Modularity

You can modularize your code by including files. This effectively "inserts" the code from the target file into your current script.

include "powerups.txt"
include "strings/common_strings.txt"

File paths are relative to data\multiplayer\megalo.

Autocomplete Notation

When typing actions, pay attention to the notations at the bottom of the window:

  • [Square Brackets]: Optional parameters.
  • {Curly | Braces}: A choice between multiple options.
  • (Parentheses): Extra information.
