Engine Data & Map Permissions

Learn how to define the metadata of your gametype and set map-specific permissions.

Engine Data

Engine data defines the core metadata of your gametype, including its name, description, and category. Note that the MCC inherited its iconography from categories.

engine_data
    name "Example Gametype"
    description engine_description
    icon k_engine_icon_eightball
    category community
end

Common categories include slayer, capture_the_flag, and community.

Map Permissions

You can set map permissions to either allow or deny your gametype on certain built-in maps. These permissions do not affect Forge variants built on those maps.

map_permissions
    default true ; Allow on all built-in maps...
    exception k_map_id_zealot ; ...except for Zealot
    exception k_map_id_52_reflection ; ...and Reflection
end

Icons & Inclusions

Including engine_icons.txt and map_ids.txt is highly recommended, as it allows you to reference icons and maps by name instead of obscure numbers.
